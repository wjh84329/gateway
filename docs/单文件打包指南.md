# 单文件打包指南

本文档介绍如何将网关程序构建为单个可执行文件，便于部署和分发。

> **当前项目已验证可行的推荐方案：静态编译 Qt。**
>
> 你当前环境已经验证成功：
> - 静态 Qt 安装目录：`D:\Qt\6.11.0\msvc2022_64_static`
> - 项目静态构建输出：`D:\7xNew\gateway\out\build_static_release\支付网关.exe`
> - 结果：**`支付网关.exe` 可直接启动**

## 推荐方案：静态编译 Qt（真正单文件直运行）

如果目标是：

- 只有一个 `支付网关.exe`
- 运行时直接启动
- 不依赖外部 Qt DLL
- 不经过 Enigma 这类运行时封装/解压

那么应优先使用 **静态编译 Qt**。

### 当前已验证环境

- Qt 源码目录：`D:\qt-everywhere-src-6.11.0`
- Qt 静态构建目录：`D:\qt-build-6.11.0-static-x64`
- Qt 静态安装目录：`D:\Qt\6.11.0\msvc2022_64_static`
- gateway 静态构建目录：`D:\7xNew\gateway\out\build_static_release`

### 一键重编脚本

已提供以下脚本：

- `scripts\build_qt_static_6_11_0.bat`：一键配置 / 编译 / 安装 **Qt 6.11.0 静态版**
- `scripts\rebuild_gateway_static_release.bat`：一键使用静态 Qt **重编 gateway**（Qt 6）
- `scripts\build_qt_static_5_15.bat`：自编 **Qt 5.15.x 静态版**（`configure` + `jom`/`nmake`，用于老系统单文件）
- `scripts\rebuild_gateway_static_release_qt5.bat`：使用静态 Qt5 **重编 gateway**（输出在 `out\build_static_release_qt5`）

### 日常重编 gateway（推荐）

如果静态 Qt 已经编译并安装完成，平时只需要执行：

```batch
cd /d D:\7xNew\gateway
scripts\rebuild_gateway_static_release.bat
```

脚本会自动：

1. 加载 Visual Studio x64 编译环境
2. 定位 Visual Studio 自带 `cmake.exe`
3. 使用 `D:\Qt\6.11.0\msvc2022_64_static` 重新配置项目
4. 在 `out\build_static_release` 中编译生成 `支付网关.exe`

### 首次从零构建静态 Qt

如果静态 Qt 还没准备好，可以执行：

```batch
cd /d D:\7xNew\gateway
scripts\build_qt_static_6_11_0.bat
```

这个脚本会：

1. 加载 VS x64 环境
2. 调用 `D:\qt-everywhere-src-6.11.0\configure.bat`
3. 以 `-static -static-runtime` 配置 Qt
4. 自动执行 `cmake --build . --parallel`
5. 自动执行 `cmake --install .`

### Qt 静态编译安装全过程（完整说明）

#### 第 1 步：准备环境

建议准备好以下工具：

- Visual Studio 2026 / 2022，并安装“使用 C++ 的桌面开发”
- CMake
- Ninja
- Python
- Perl

然后打开：

- **x64 Native Tools Command Prompt for VS**

或由脚本自动加载 VS x64 编译环境。

#### 第 2 步：准备目录

如果目录还不存在，可以手动创建：

```batch
mkdir D:\qt-build-6.11.0-static-x64
mkdir D:\Qt\6.11.0
```

然后进入构建目录：

```batch
cd /d D:\qt-build-6.11.0-static-x64
```

#### 第 3 步：配置 Qt 静态构建

执行：

```batch
D:\qt-everywhere-src-6.11.0\configure.bat -prefix D:\Qt\6.11.0\msvc2022_64_static -release -static -static-runtime -submodules qtbase,qttools -nomake examples -nomake tests -opensource -confirm-license -cmake-generator Ninja
```

参数含义：

- `-static`：生成静态 Qt
- `-static-runtime`：尽量把 MSVC 运行库也静态链接进去
- `-submodules qtbase,qttools`：只编当前项目需要的核心模块，缩短编译时间
- `-nomake examples -nomake tests`：不编示例和测试
- `-cmake-generator Ninja`：使用 Ninja 构建

#### 第 4 步：编译 Qt

执行：

```batch
cmake --build . --parallel
```

说明：

- 首次构建耗时较长，可能几十分钟到几小时
- 看到类似 `Linking CXX executable qtbase\bin\qtdiag.exe` 且没有错误时，通常表示编译成功

#### 第 5 步：安装 Qt

执行：

```batch
cmake --install .
```

安装完成后，Qt 会被安装到：

```text
D:\Qt\6.11.0\msvc2022_64_static
```

#### 第 6 步：检查安装结果

至少确认这些目录存在：

```text
D:\Qt\6.11.0\msvc2022_64_static\lib
D:\Qt\6.11.0\msvc2022_64_static\lib\cmake\Qt6
D:\Qt\6.11.0\msvc2022_64_static\plugins\platforms
D:\Qt\6.11.0\msvc2022_64_static\plugins\sqldrivers
```

你当前环境中还已经确认存在：

- `platforms\qwindows.lib`
- `sqldrivers\qsqlodbc.lib`

这对当前 `gateway` 项目很重要。

### 手动命令（已验证）

#### 1）静态编译 Qt 6.11.0

在 **x64 Native Tools Command Prompt for VS** 中执行：

```batch
cd /d D:\qt-build-6.11.0-static-x64
D:\qt-everywhere-src-6.11.0\configure.bat -prefix D:\Qt\6.11.0\msvc2022_64_static -release -static -static-runtime -submodules qtbase,qttools -nomake examples -nomake tests -opensource -confirm-license -cmake-generator Ninja
cmake --build . --parallel
cmake --install .
```

#### 2）使用静态 Qt 构建 gateway

```batch
cd /d D:\7xNew\gateway
cmake -S . -B out\build_static_release -G Ninja -DCMAKE_PREFIX_PATH=D:\Qt\6.11.0\msvc2022_64_static -DCMAKE_BUILD_TYPE=Release
cmake --build out\build_static_release
```

### 输出文件

构建成功后，输出文件为：

- `D:\7xNew\gateway\out\build_static_release\支付网关.exe`

该文件已经在当前环境下验证为：**可以直接启动**。

### 注意事项

1. **Ninja 是单配置生成器**
   - 使用 Ninja 时，`--config Release` 基本不起决定作用
   - 必须在配置阶段指定：
     ```batch
     -DCMAKE_BUILD_TYPE=Release
     ```

2. **不要混用 Debug / Release**
   - 静态 Qt 本次是按 `Release` 编译的
   - gateway 也必须按 `Release` 编译
   - 否则会出现 `_ITERATOR_DEBUG_LEVEL` 和 `RuntimeLibrary` 不匹配

3. **不要依赖普通 cmd 的 PATH**
   - 你之前已经遇到过普通环境下 `cmake` 不在 PATH 的问题
   - 现在提供的 bat 脚本会优先自动加载 VS 环境并寻找 VS 自带 `cmake.exe`

4. **Qt Sql / ODBC 已纳入当前方案关注范围**
   - 项目中 `rechargeprocessor.cpp` 使用了 `QODBC`
   - 当前静态 Qt 安装中已确认存在：`sqldrivers\qsqlodbc.lib`

## 方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **Enigma Virtual Box** | 免费、简单、上手快 | 运行时存在封装/解包行为 | 适合临时分发、快速验证 |
| **静态编译 Qt** | 真正单文件、无需解压、直接运行 | 首次成本高、编译慢 | **推荐用于当前项目正式单文件方案** |
| **BoxedApp Packer** | 商业支持、功能强大 | 需要购买许可证 | 企业级部署 |

## 备选方案：Enigma Virtual Box

### 准备工作

1. **下载安装 Enigma Virtual Box**
   - 下载地址：https://enigmaprotector.com/en/files/enigmavb.html
   - 你的当前安装路径：`D:\Program Files (x86)\Enigma Virtual Box\`

2. **确保 Qt 环境变量已设置**
   - 设置 `Qt5_DIR` 或 `Qt6_DIR` 环境变量
   - 例如：`Qt5_DIR=C:\Qt\5.15.2\msvc2019_64`

### 使用方法

#### 方法一：使用批处理脚本

```batch
cd d:\7xNew\gateway
scripts\build_single_file.bat
```

#### 方法二：使用 PowerShell 脚本

```powershell
cd d:\7xNew\gateway
scripts\build_single_file.ps1
```

#### 方法三：手动执行

1. **构建项目**
   ```batch
   cmake -S . -B out\build
   cmake --build out\build --config Release
   ```

2. **部署 Qt 依赖**
   ```batch
   windeployqt --release --no-compiler-runtime --no-translations --no-opengl-sw --dir out\build\Release\qt_plugins out\build\Release\支付网关.exe
   ```

3. **创建 Enigma Virtual Box 配置文件** (`evb_config.xml`)
   ```xml
   <?xml version="1.0" encoding="utf-8"?>
   <enigmavb>
       <output>out\build\Release\cs_single.exe</output>
       <input>out\build\Release\支付网关.exe</input>
       <packfiles>
           <pack>
               <folder>out\build\Release\qt_plugins</folder>
               <subfolders>true</subfolders>
               <root>out\build\Release</root>
           </pack>
       </packfiles>
       <compress>true</compress>
       <compresslevel>9</compresslevel>
       <temppath>%TEMP%</temppath>
       <executionmode>0</executionmode>
   </enigmavb>
   ```

4. **打包**
   ```batch
   "D:\Program Files (x86)\Enigma Virtual Box\enigmavb.exe" pack evb_config.xml
   ```

### 输出

打包完成后，将在 `out\build\Release\` 目录生成：
- `支付网关.exe` - 原始可执行文件（需要依赖目录）
- `cs_single.exe` - 单文件版本（推荐分发）

## Qt 5.15 静态（自编单文件，兼容 Win7 / Server 2008 R2）

若必须 **不换用户系统** 且要 **单文件 exe**，应使用 **Qt 5.15.x 静态库** 自行编译（官方不提供 Windows 静态成品安装包，只有源码归档）。

与 **Qt 6** 的区别：**Qt 5.15** 在 Windows 上使用 **`configure.bat` + `jom` / `nmake`** 构建，**不是** Qt 6 那种 `configure.bat -cmake-generator Ninja` + `cmake --build` 流程。

### 源码下载

- 官方归档目录：https://download.qt.io/archive/qt/5.15/
- 进入具体版本（如 `5.15.15/single/`），下载 **`qt-everywhere-opensource-src-5.15.15.zip`**，解压到与脚本一致的目录（默认脚本使用 `D:\qt-everywhere-src-5.15.15`）。

### 一键脚本（推荐）

1. 编辑 `scripts\build_qt_static_5_15.bat` 顶部三行路径：`QT_SOURCE_DIR`、`QT_BUILD_DIR`、`QT_INSTALL_DIR`（须与解压位置一致）。
2. 在 **x64 Native Tools Command Prompt for VS** 或普通 cmd 中执行：
   ```batch
   cd /d D:\7xNew\gateway
   scripts\build_qt_static_5_15.bat
   ```
3. Qt 安装完成后，编辑 `scripts\rebuild_gateway_static_release_qt5.bat` 中的 `STATIC_QT_DIR`，使其与上一步 `QT_INSTALL_DIR` 一致，然后：
   ```batch
   scripts\rebuild_gateway_static_release_qt5.bat
   ```
4. 输出 exe 路径：`out\build_static_release_qt5\支付网关.exe`（需 gateway 工程已能使用 Qt5 编过；若仅有 Qt6 专有 API，需先按 Qt5 做小改动）。

### 环境说明

- **编译器**：建议 **VS 2019** 或 **VS 2022**（与 Qt 5.15 小版本匹配关系见 Qt 文档）；脚本会尝试自动定位 `VsDevCmd.bat`。
- **jom**：强烈建议安装并加入 PATH，否则脚本会退化为 **nmake**，耗时会显著变长。说明：https://wiki.qt.io/Jom
- **ODBC**：脚本已加 **`-sql-odbc`**，与当前项目 `QODBC` 一致。
- **静态链接 SQL 驱动**：若链接阶段报缺 ODBC 插件，可能需在工程或 `main.cpp` 中按 Qt 文档为静态插件增加 `Q_IMPORT_PLUGIN` / CMake 导入（与 Qt 6 静态细节不同，以实际链接错误为准）。

### 注意事项

- 磁盘与时间与 Qt 6 类似：预留 **约 20GB+** 空间，首次全量编译常需 **数小时**（跳过 `qtwebengine` 后仍较长）。
- 单文件体积通常为 **几十 MB 级**，属静态链接正常现象。
- 遵守 Qt **LGPL / 商业许可** 义务（静态链接时尤需注意分发合规）。

## 常见问题

### Q: 打包后程序无法启动

A: 可能是缺少 Visual C++ 运行库，建议：
- 安装 [Visual C++ Redistributable](https://aka.ms/vs/17/release/vc_redist.x64.exe)
- 或者在打包时包含运行时：移除 `--no-compiler-runtime` 参数

### Q: 如何减小单文件大小

A: 可以：
- 移除不需要的 Qt 模块
- 使用 UPX 压缩（在 Enigma Virtual Box 配置中设置 `<upx>true</upx>`）
- 只打包必需的插件

### Q: 打包后杀毒软件报毒

A: Enigma Virtual Box 打包的程序可能被误报，可以：
- 使用代码签名证书签名
- 向杀毒软件厂商提交白名单
- 使用其他打包方案
